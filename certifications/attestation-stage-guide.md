## ⏱️ Durée du Stage

**Bien remplir les champs :**
- **Date début :** Format jour/mois/année (ex: 05/09/2024)
- **Date fin :** Format jour/mois/année (ex: 25/10/2024)
- **Nombre commun de semaines :** Calculer = (fin - début) / 7 jours
- **Nombre de jours/semaines de mois :**
  - Si stage sur 2 mois : ex "2 mois"
  - Si stage sur 8 semaines : ex "8 semaines"

**Calcul exemple :**
- Début : 05/09/2024
- Fin : 25/10/2024
- Durée : 51 jours = 7.3 semaines ≈ **7 semaines**

```
Première semaine du stage

CONTEXTE :
- Accueil en entreprise [nom], équipe de [X] développeurs

1. Comprendre l'existant technique (stack : Node.js, React, PostgreSQL)
```

```
OBJECTIFS :

✅ Versionning Git
   - 35 commits sémantiques

   - Documentation API (Swagger)

COMPÉTENCES VALIDÉES :

✅ BLOC 2.3 — Développer application (démo MVP réussie)
```

## ✍️ Conseils de Rédaction

### Ton & Langage

✅ **À faire :**
- Professionnel mais accessible
- Action-oriented (verbes d'action : "développer", "implémenter", "analyser")
- Spécifique (nombres, technologies, résultats concrets)

❌ **À éviter :**
- Trop informel ("super", "génial")
- Vague ("j'ai travaillé sur des trucs")
- Trop technique sans explications

### Structure d'une Semaine Idéale

```
SEMAINE N : [Titre thématique]

CONTEXTE : Où étais-je ? Que voulait l'entreprise ?
↓
OBJECTIFS : Qu'avais-je à faire cette semaine ?
↓
RÉALISATIONS : Qu'ai-je réellement fait ?
↓
PREUVES TECHNIQUES : Code, commits, tests
↓
COMPÉTENCES VALIDÉES : Quelles compétences BTS ?
```
